# Suave Dynamic Showreel Stack

## Primary Use Case

This animation is a premium vertical showreel-style card composition built for cinematic interface moments, editorial motion layouts, luxury SaaS visuals, social promo loops, and high-end modular motion systems.

It is designed for scenarios where multiple visual panels need to exist in one composition while remaining lightweight, elegant, and easy to swap dynamically.

This is not a text-led animation.
This is a visual-structure-led animation.

- two upper support cards

## Motion Sequence

### 1. Initial Arrival

The three cards animate into position with:

- soft directional offset
- slight rotation correction
- opacity resolve
- subtle scale settle

The motion is intentionally controlled and avoids over-animated behavior.

### 2. Internal Asset Resolve

Once the card shells settle into place, their internal slot content resolves gently.

Examples include:

- wave forms
- bars
- chip-like geometric icons
- badge forms
- ring clusters
- orbit pack structures
- portrait placeholders
- circular emblem / coin forms

This internal motion is kept minimal to preserve smooth rendering.

### 3. Ambient Float

After settling, the composition enters a restrained idle state:

- each card drifts gently
- the background orb softly breathes
- the frame slightly expands/contracts
- the ember accent subtly lifts and settles

This keeps the scene alive without introducing noise.

### 4. State Swap

The animation then transitions into the next visual state.

The outgoing cards:

- shift slightly off-axis
- reduce opacity
- scale down subtly

The next state is injected immediately after the outgoing state clears.

The new state then resolves with a soft entrance rather than a harsh snap.

### 5. Loop Continuation

The system loops through configured states using the same motion language.

This makes the animation feel like a reusable motion shell rather than a fixed one-shot composition.

## Asset Slot Philosophy

This animation should be treated as a reusable slot-based motion container.

The motion shell controls:

- position
- hierarchy
- entrance
- exit
- idle motion
- overall composition

The inserted asset controls:

- what appears inside the slot
- whether the slot contains a built-in primitive
- whether it contains an image
- whether it contains custom HTML

This separation is important because it allows the animation to evolve into a retrieval-ready motion beat inside a larger system.

The slot architecture supports future extension toward:
- static images
- transparent PNG cutouts
- SVG artwork
- nested animated HTML modules
- lightweight micro-interaction content

## Rendering / Performance Notes

This version was designed specifically to improve smoothness and reduce visible lag.

Performance-conscious decisions include:

- mostly transform-based motion
- opacity-based transitions
- minimal filter usage
- no animated blur choreography
- no backdrop-filter
- restrained shadow complexity
- lightweight internal asset geometry
- minimal text rendering overhead

This makes it significantly more preview-friendly than heavier browser-rendered motion systems.
